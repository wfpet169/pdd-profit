# 拼多多利润核算

一款专为拼多多商家设计的电商利润核算管理工具，帮助商家快速导入订单数据、分析利润情况、管理商品成本。

## 功能特性

### 1. 数据导入
- 支持导入 Excel (.xlsx, .xls) 和 CSV 格式的订单数据
- 下载拼多多对账报表：拼多多后台发货管理 > 订单查询 > 全部订单 > 按订单下单时间批量导出 > 生成对账报表 > 下载报表上传即可
- 自动识别常见表头字段，兼容多种Excel模板格式
- 支持订单号、商品规格、数量、单价、总金额、运费、平台服务费、成本价等字段

### 2. 利润概览（首页仪表盘）
- **总收入**：统计不含已取消订单的总收入
- **订单数量**：显示不含已取消订单的数量
- **总利润**：计算实际利润（收入 - 成本 - 平台服务费 - 运费 - 其他费用）
- **利润率**：利润与收入的百分比
- **退货率分析**：按状态分类统计退款订单
  - 未发货退款
  - 已发货退款
  - 已收货退款
  - 未付款已取消
- **退款总金额**：统计三种退款类型的金额总和
- **利润趋势图**：可视化展示每日利润和收入变化
- **热销商品排行**：展示销售额TOP10商品

### 3. 订单管理
- 完整的订单列表展示
- 多维度筛选功能：
  - 按商品规格筛选（支持多选下拉）
  - 按订单日期筛选
  - 按金额范围筛选
  - 按成本范围筛选
  - 按利润范围筛选
  - 按订单状态筛选
- 支持导出数据到Excel

### 4. 成本管理
- 商品规格成本配置
- 支持设置：
  - 商品规格（需与导入的Excel中商品规格名称完全一致）
  - 成本价（元/件）
  - 平台服务费（%）
  - 快递成本
  - 其他成本（包装、人工等）
- 自动匹配订单计算利润

## Excel 数据格式要求

导入的Excel文件需包含以下字段（表头）：

| 字段 | 可用表头名称 |
|------|-------------|
| 订单号 | 订单号、订单编号、订单号/号码 |
| 商品规格 | 商品名称、商品、商品规格、产品名称 |
| 数量 | 数量、商品数量(件)、件数 |
| 单价 | 单价、单价(元)、单价(分) |
| 总金额 | 总价、总金额、商家实收金额(元)、订单金额、金额 |
| 运费 | 运费、快递费、物流费 |
| 平台服务费 | 平台服务费、技术服务费 |
| 成本价 | 成本价、成本 |
| 其他费用 | 其他费用、其他 |
| 订单日期 | 订单日期、日期、下单时间、创建时间、订单成交时间 |
| 订单状态 | 订单状态、状态 |

### 订单状态说明

系统支持的订单状态（Excel中的原始值）：
- **未发货**：正常待发货订单
- **未发货，退款成功**：未发货时申请退款
- **已发货，退款成功**：已发货后申请退款
- **已收货，退款成功**：已收货后申请退款（退货）
- **已取消**：拍下未付款已取消的订单

## 技术栈

- **前端框架**: React 19
- **语言**: TypeScript
- **构建工具**: Vite 8
- **路由**: React Router DOM
- **图表**: Recharts
- **数据处理**: SheetJS (xlsx)
- **图标**: Lucide React
- **存储**: IndexedDB (本地浏览器数据库)

## 注意事项

1. 商品规格名称必须与导入的Excel中的商品规格完全一致，才能正确匹配成本配置
2. 订单状态使用Excel中的原始值，系统会根据状态自动区分正常订单和退款/取消订单
3. 所有数据存储在浏览器本地IndexedDB中，不会上传到服务器
